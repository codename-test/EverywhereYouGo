# 推送模板编写指南

> 📌 快速跳转：[AI 提示词](#ai-prompt)

## 概述

模板（Template）负责把解析器输出的结构化字段，渲染成适合具体通道阅读的文本。每个通道绑定可以单独指定模板，因此同一份数据可以为不同平台呈现不同格式。

渲染时机：

```
解析器输出字典 → 路由匹配 → 模板渲染（本文档） → 通道 send()
```

## 模板引擎

EGo 支持两种引擎，创建模板时选择：

| 引擎 | 能力 | 适用场景 |
|------|------|----------|
| **Simple** | 仅变量替换 `{var}` | 简单固定格式 |
| **Jinja2** | 变量、条件、循环、过滤器 | 需要判空、分支、复杂排版 |

## 可用变量

变量来自解析器返回的字典键。常见变量：

| 变量 | 含义 |
|------|------|
| `title` | 消息标题 |
| `content` | 消息正文 |
| `url` | 来源链接 |
| `author` | 内容作者 |
| `timestamp` | 事件时间 |

此外还有系统字段 `_trace_id`（消息追踪号）。具体可用变量取决于你绑定的解析器输出了哪些键——在数据源配置的「可用变量」面板可以查看。

## Simple 引擎

用 `{变量名}` 做直接替换：

```
**{title}**

{content}

{url}
```

Simple 引擎不支持条件判断，空变量会渲染成空字符串。

## Jinja2 引擎

### 变量

```jinja2
{{ title }}
```

### 条件判断

字段可能为空时，用条件包裹避免输出空行：

```jinja2
**{{ title }}**

{{ content }}

{% if url %}
查看详情: {{ url }}
{% endif %}

{% if author %}
作者: {{ author }}
{% endif %}
```

### 过滤器

```jinja2
{{ content | truncate(200) }}     {# 截断到 200 字符 #}
{{ title | upper }}               {# 转大写 #}
```

### 循环

解析器输出列表字段时可以遍历：

```jinja2
{% for tag in tags %}
#{{ tag }}
{% endfor %}
```

## 完整示例

一个兼顾各字段判空的媒体通知模板（Jinja2）：

```jinja2
🎬 **{{ title }}**

{{ content }}

{% if item_type %}类型: {{ item_type }}{% endif %}
{% if author %}艺术家: {{ author }}{% endif %}
{% if url %}
▶ 立即观看: {{ url }}
{% endif %}
```

## 最佳实践

- 字段可能缺失时用 Jinja2 的 `{% if %}` 判空，避免空行/空链接
- 模板保持简洁，复杂逻辑放到解析器里处理
- 用数据源的「测试解析 / 测试推送」配合真实样本验证输出
- 不同通道对 Markdown 的支持程度不同（如 Bark 不支持复杂 Markdown），按目标平台调整格式

## 常见问题

**Q：变量名写错了会怎样？**
A：Simple 引擎会原样保留 `{xxx}` 文本；Jinja2 默认渲染成空字符串。请对照解析器输出检查变量名。

**Q：怎么知道有哪些变量可用？**
A：在数据源配置弹窗中选择解析器后点击「变量」按钮，会列出该解析器输出的所有键。

**Q：模板里能写 Python 代码吗？**
A：不能。Jinja2 沙箱只允许模板语法和内置过滤器，不能执行任意代码。
