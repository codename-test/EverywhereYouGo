{{ _("sdk.overview_title") }}

通道插件（Channel Plugin）是 EGo 推送系统的最终执行层。它负责将解析并模板化后的消息，发送到具体的目标平台（如 Bark、Telegram、邮件等）。

A Channel Plugin is the final delivery layer of the EGo push system. It is responsible for sending parsed and templated messages to a specific target platform (e.g., Bark, Telegram, Email, etc.).

{{ _("sdk.overview_arch") }}

RSS / JSON / HTML → {{ _("sdk.arch_parser") }} → {{ _("sdk.arch_template") }} → {{ _("sdk.arch_channel") }} → {{ _("sdk.arch_target") }}

数据流经以下阶段：

  1. 数据源（Source）：提供原始数据（RSS、网页、API 等）
  2. 解析器（Parser）：从原始数据中提取标题和内容
  3. 模板（Template）：将提取的内容格式化为消息文本
  4. 通道插件（Channel Plugin）：将消息推送到目标平台
  5. 目标平台：用户最终收到通知的设备或服务

Data flows through the following stages:

  1. Source: Provides raw data (RSS, web pages, APIs, etc.)
  2. Parser: Extracts titles and content from raw data
  3. Template: Formats extracted content into message text
  4. Channel Plugin: Pushes the message to the target platform
  5. Target: The device or service where the user receives the notification

Below is the full bark.py implementation with detailed comments explaining each part:

#!/usr/bin/python3
# -*- coding: UTF-8 -*-
"""Bark iOS Push Notification Channel
Bark iOS 推送通道插件
"""

import requests
import log
from channel_loader import BaseChannel


class Channel(BaseChannel):
    # ── Class Attributes / 类属性 ──────────────────────────────
    # CHANNEL_TYPE: unique identifier, used in database and routing
    # CHANNEL_TYPE: 唯一标识符，用于数据库和消息路由
    CHANNEL_TYPE = "bark"

    # CHANNEL_NAME: human-readable display name shown in WebUI
    # CHANNEL_NAME: 在 WebUI 中显示的名称
    CHANNEL_NAME = "Bark"

    # ── Initialization / 初始化 ────────────────────────────────
    def __init__(self, config: dict):
        # Always call super().__init__() first
        # 必须先调用 super().__init__()
        super().__init__(config)

        # Read config values with safe defaults
        # 使用安全默认值读取配置
        self.server_url = config.get("server_url", "https://api.day.app")
        self.device_key = config.get("device_key", "")

    # ── Send Message / 发送消息 ────────────────────────────────
    def send(self, title: str, content: str) -> tuple:
        """Send a push notification via Bark API
        通过 Bark API 发送推送通知

        Returns: (success: bool, error_message: str)
        返回: (是否成功, 错误信息)
        """
        # Validate required config
        # 验证必需的配置项
        if not self.device_key:
            return False, "device_key is empty"

        try:
            # Make HTTP POST request to Bark API
            # 向 Bark API 发送 HTTP POST 请求
            resp = requests.post(
                f"{self.server_url}/{self.device_key}",
                json={
                    "title": title,
                    "body": content,
                    "group": "EverywhereYouGo"   # Group notifications
                },
                timeout=15  # Reasonable timeout / 合理的超时时间
            )

            # Parse response and check success
            # 解析响应并检查是否成功
            body = resp.json()
            if body.get("code") == 200:
                return True, ""

            # Extract error message from response
            # 从响应中提取错误信息
            errmsg = body.get("message", "unknown error")
            log.logger.error(f"Bark send: {errmsg}")
            return False, errmsg

        except Exception as e:
            # Catch all exceptions, log and return error
            # 捕获所有异常，记录日志并返回错误
            log.logger.error(f"Bark send: {e}")
            return False, str(e)

    # ── Connectivity Test / 连通性测试 ─────────────────────────
    def test(self) -> bool:
        """Test if the Bark channel is working
        测试 Bark 通道是否可用
        """
        if not self.device_key:
            return False
        try:
            resp = requests.post(
                f"{self.server_url}/{self.device_key}",
                json={
                    "title": "EverywhereYouGo 测试",
                    "body": "通道测试成功！",
                    "group": "EverywhereYouGo"
                },
                timeout=15
            )
            return resp.json().get("code") == 200
        except Exception:
            return False

Write your Channel class, inheriting from BaseChannel, implementing the send() and test() methods.

import requests
import log
from channel_loader import BaseChannel


class Channel(BaseChannel):
    CHANNEL_TYPE = "my_channel"
    CHANNEL_NAME = "My Channel"

    CONFIG_FIELDS = [
        {
            "name": "webhook_url",
            "type": "text",
            "label": "Webhook URL",
            "label_zh": "Webhook 地址",
            "desc": "Your webhook URL",
            "desc_zh": "你的 Webhook 地址",
            "placeholder": "https://...",
            "required": True,
            "default": ""
        }
    ]

    def __init__(self, config: dict):
        super().__init__(config)
        self.api_key = config.get("api_key", "")

    def send(self, title: str, content: str) -> tuple:
        try:
            resp = requests.post(
                "https://api.example.com/push",
                json={"key": self.api_key, "title": title, "body": content},
                timeout=15
            )
            if resp.status_code == 200:
                return True, ""
            return False, f"HTTP {resp.status_code}"
        except Exception as e:
            return False, str(e)

    def test(self) -> bool:
        ok, _ = self.send("EGo Test", "Channel test message")
        return ok

{{ _("sdk.tips_requests") }}

使用 requests 库进行 HTTP 调用。该库已在项目中预装，无需额外安装。

Use the requests library for HTTP calls. It is already available in the project, no extra installation needed.

{{ _("sdk.tips_exceptions") }}

始终在 send() 和 test() 中用 try/except 捕获异常，并返回有意义的错误信息。未捕获的异常会导致推送静默失败。

Always wrap send() and test() logic in try/except and return meaningful error messages. Uncaught exceptions will cause silent push failures.

def send(self, title: str, content: str) -> tuple:
    try:
        # your logic here
        ...
    except Exception as e:
        log.logger.error(f"MyChannel send: {e}")
        return False, str(e)
{{ _("sdk.tips_timeout") }}

为所有 HTTP 请求设置合理的超时时间（推荐 15 秒）。过长的超时会导致消息队列阻塞。

Set reasonable timeouts for all HTTP requests (15 seconds recommended). Excessively long timeouts can block the message queue.

{{ _("sdk.tips_test_real") }}

test() 方法应该发送一条真实的测试消息，而不是仅仅检查网络连接。这样用户可以确认整个推送链路是否正常。

The test() method should send a real test message, not just check network connectivity. This lets users confirm the entire push pipeline works end-to-end.

{{ _("sdk.tips_logging") }}

使用 log.logger 记录关键操作和错误。日志可以在 WebUI 的"日志"页面查看，便于调试。

Use log.logger to log key operations and errors. Logs are viewable in the WebUI "Logs" page, making debugging easier.

import log

log.logger.info("MyChannel: message sent successfully")
log.logger.error(f"MyChannel: failed - {error_msg}")
